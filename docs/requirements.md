# 요구사항 분석

## 1. 프로젝트 개요
- 프로젝트명: Expense Tracker
- 유형: 웹 애플리케이션
- 목적: 개인 수입/지출을 간편하게 기록하고 잔액을 확인하는 도구 제공

## 2. 기능 요구사항 (Functional Requirements)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-01 | 사용자는 수입 내역을 추가할 수 있다 | 상 |
| FR-02 | 사용자는 지출 내역을 추가할 수 있다 | 상 |
| FR-03 | 사용자는 거래 내역을 삭제할 수 있다 | 상 |
| FR-04 | 시스템은 총수입, 총지출, 잔액을 자동 계산한다 | 상 |
| FR-05 | 시스템은 입력값(금액, 항목명)의 유효성을 검증한다 | 중 |
| FR-06 | 사용자는 거래 내역 목록을 조회할 수 있다 | 상 |

## 3. 비기능 요구사항 (Non-functional Requirements)

| ID | 요구사항 |
|----|---------|
| NFR-01 | 별도 설치 없이 브라우저에서 즉시 실행 가능해야 한다 |
| NFR-02 | 입력 오류 시 사용자에게 명확한 피드백을 제공해야 한다 |
| NFR-03 | UI는 직관적이고 단순해야 한다 |

## 4. 유스케이스 (Use Case)

**액터:** 사용자  
**유스케이스 목록:**
- UC-01: 수입 추가
- UC-02: 지출 추가
- UC-03: 거래 삭제
- UC-04: 잔액 확인

## 5. 제약사항
- 순수 HTML/CSS/JavaScript로 구현 (프레임워크 미사용)
- 데이터는 세션 내 메모리에만 저장 (DB 미사용)
