# 테스트 문서

## 1. 테스트 개요
구현된 Expense Tracker의 기능이 요구사항대로 동작하는지 검증한다.
브라우저 기반 수동 테스트(Manual Test)로 수행한다.

## 2. 테스트 케이스

| TC ID | 테스트 항목 | 입력 | 기대 결과 | 실제 결과 | 결과 |
|-------|-----------|------|----------|----------|------|
| TC-01 | 수입 추가 | "월급", 3000000, 수입 | 총수입 3,000,000 / 잔액 +3,000,000 | 정상 반영 | PASS |
| TC-02 | 지출 추가 | "점심", 8000, 지출 | 총지출 8,000 / 잔액 2,992,000 | 정상 반영 | PASS |
| TC-03 | 거래 삭제 | 항목 옆 [삭제] 클릭 | 해당 항목 삭제 + 잔액 재계산 | 정상 동작 | PASS |
| TC-04 | 빈 항목명 입력 | "", 1000 | "항목명을 입력하세요." 에러 표시 | 에러 표시됨 | PASS |
| TC-05 | 음수 금액 입력 | "월급", -100 | "금액은 0보다 큰 숫자여야 합니다." 에러 | 에러 표시됨 | PASS |

## 4. 테스트 결과 요약
- 총 테스트 케이스: 9개

- 실패: 0개
- 모든 기능 요구사항이 정상적으로 동작함을 확인하였다.

## 5. 추가 검토 필요 사항
기능 테스트는 모두 통과하였으나, 다음 사항은 별도 품질 점검에서 추가 검토 예정이다.
- 사용자 입력값의 보안성 (XSS 등)
- 데이터 영속성
- 코드 가독성 및 유지보수성
