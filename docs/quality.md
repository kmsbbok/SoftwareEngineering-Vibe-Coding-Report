# 품질 관리 문서

## 1. 품질 관리 개요
기능 테스트(test.md) 통과 이후, 비기능적 품질 측면에서 코드를 추가 점검하였다.
보안성, 신뢰성, 유지보수성 관점에서 개선점을 도출하고 반영하였다.

## 2. 발견된 품질 이슈

### 이슈 1: 사용자 입력의 XSS 취약점
- 발견 경위: 코드 리뷰 중 renderList() 함수에서 innerHTML 사용 확인
- 재현 방법: 항목명 입력란에 `<img src=x onerror="alert('XSS')">` 입력 후 추가
- 결과: 입력된 스크립트가 실행되어 alert 창이 표시됨
- 원인: innerHTML은 문자열을 HTML로 해석하여 임의 코드 실행을 허용함
- 조치: 항목명 출력 시 HTML escape 처리 함수(escapeHtml)를 적용
- 재검증: 동일 입력 후 스크립트가 실행되지 않고 문자열 그대로 표시됨을 확인

## 3. 적용한 품질 관리 기법

### 코드 리뷰
AI의 도움을 받아 보안 관점에서 코드를 점검하였다.
기능 테스트만으로는 발견되지 않는 보안 취약점을 코드 리뷰를 통해 식별하였다.

### 회귀 테스트 (Regression Test)
보안 수정 이후, 기존 테스트 케이스(TC-01 ~ TC-09)를 재수행하여
수정이 기존 기능에 영향을 주지 않았음을 확인하였다.

| TC ID | 수정 전 | 수정 후 |
|-------|--------|--------|
| TC-01 ~ TC-09 | PASS | PASS |

### 품질 속성 점검
- 보안성: XSS 취약점 제거
- 신뢰성: 모든 기능 요구사항 정상 동작 유지
- 유지보수성: escape 처리를 별도 함수로 분리하여 재사용성 확보
