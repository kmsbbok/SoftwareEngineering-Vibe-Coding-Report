# SoftwareEngineering-Vibe-Coding-Report

## 프로젝트 개요

본 저장소는 소프트웨어공학 과제인  
**"프로세스에 입각한 바이브코딩의 효과 분석"** 보고서 작성을 위한 저장소이다.

본 프로젝트에서는 AI를 활용한 바이브코딩 방식으로 간단한 소프트웨어를 개발하고,  
요구사항 분석, 설계, 구현, 테스트, 품질 관리에 이르는 소프트웨어 개발 프로세스를 적용한다.

## 개발 예정 소프트웨어

- 프로젝트명: Expense Tracker
- 개발 형태: 웹 애플리케이션
- 주요 기능:
  - 수입 및 지출 내역 추가
  - 거래 내역 삭제
  - 총수입, 총지출, 잔액 계산
  - 입력값 검증

## AI 활용 계획

본 프로젝트에서는 ChatGPT를 바이브코딩 도구로 활용한다.

AI는 다음 과정에서 보조 도구로 사용한다.

- 프로젝트 주제 선정
- 요구사항 분석 초안 작성
- 설계 방향 정리
- 초기 코드 생성
- 오류 수정 방향 제안
- 테스트 케이스 작성
- 품질 개선점 검토

## 프로세스 적용 계획

본 프로젝트는 다음 순서로 진행한다.

1. 요구사항 분석
2. 설계
3. 구현
4. 테스트
5. 품질 관리
6. Lessons Learned 정리
7. 최종 보고서 작성

## 진행 기록 방식

본 프로젝트는 개인 단위로 수행한다.  
따라서 팀 회의록 대신 GitHub commit 기록, 작업 로그, AI 프롬프트 기록을 통해  
프로세스 적용 과정을 증빙한다.

## 1일차 작업 기록

- 과제 요구사항 확인
- GitHub 저장소 생성
- 개발 주제 후보 검토
- Expense Tracker를 개발 주제로 선정
- AI 활용 방식 및 프로세스 적용 방향 정리

## 2일차 작업 기록

- 요구사항 분석 수행
- 기능 요구사항 6개, 비기능 요구사항 3개 도출
- 유스케이스 4종 정리
- 제약사항 정의 (프레임워크 미사용, 메모리 저장 방식)
- requirements.md 문서화
- UI 와이어프레임 시안 작성 (HTML 기반)
- wireframe.html 및 wireframe.png 생성
- 아키텍처 및 모듈 구성 설계 (UI, Transaction, Calculation, Validation 모듈)
- 데이터 구조 정의 (id, type, description, amount)
- 주요 함수 5종 명세 (addTransaction, deleteTransaction, calculateBalance, validateInput, renderList)
- 요구사항 추적 매트릭스(RTM) 작성
- 설계 결정 사항 및 적용 설계 원칙(SRP, DRY, 결합도/응집도) 정리
- design.md 문서화
